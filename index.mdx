Gardener is a Kubernetes-native cluster management platform that lets you provision, operate, and scale Kubernetes clusters across any cloud provider or on-premises infrastructure. By hosting cluster control planes inside Kubernetes itself — the "Kubernetes in Kubernetes" pattern — Gardener delivers homogeneous, conformance-tested clusters with unified configuration across AWS, Azure, GCP, OpenStack, and more.

## How Gardener works

Gardener mirrors the Kubernetes architecture itself. Just as Kubernetes manages containers on nodes, Gardener manages Kubernetes clusters on seed clusters — using the same declarative, controller-based patterns you already know.

| Gardener Component | Kubernetes Equivalent | Role |
|---|---|---|
| `gardener-apiserver` | `kube-apiserver` | Exposes the Gardener API |
| `gardener-controller-manager` | `kube-controller-manager` | Reconciles Gardener resources |
| `gardener-scheduler` | `kube-scheduler` | Assigns shoots to seeds |
| `gardenlet` | `kubelet` | Agent running on each seed cluster |
| Seed cluster | Worker node | Hosts shoot control planes |
| Shoot cluster | Pod | End-user Kubernetes cluster |

<Steps>
  <Step title="Deploy the Gardener Operator">
    Install the `gardener-operator` on an existing Kubernetes runtime cluster. It manages the virtual garden cluster and all control plane components.
  </Step>
  <Step title="Register a seed cluster">
    Deploy a `gardenlet` to register a seed cluster. The seed hosts the control planes of your shoot clusters as native Kubernetes workloads.
  </Step>
  <Step title="Configure cloud providers">
    Install provider extensions (AWS, Azure, GCP, OpenStack, etc.) and create `CloudProfile` resources describing available machine types and OS images.
  </Step>
  <Step title="Create shoot clusters">
    Apply a `Shoot` resource to provision a fully managed Kubernetes cluster. Gardener handles etcd, kube-apiserver, updates, backups, and monitoring automatically.
  </Step>
</Steps>
